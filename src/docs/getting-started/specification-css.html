CSS 编码规范

此为前端开发团队遵循和约定的 CSS 编码规范，意在提高代码的规范性和可维护性。

代码风格

  • 代码应该符合 CSS 语法有效性，可以使用 W3C CSS validator 工具来验证。

  • ID 和 Class 应该按照元素功能命名，不应该按照元素表现命名，命名应该含义清晰。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad: 含义不清 */
 #yee-1901 {}

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad: 表现化 */
 .button-green {}
 .clear {}

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good: 功能化 */
 #gallery {}
 #login {}
 .video {}

  • ID 和 Class 命名应该在保持含义清晰的前提下尽可能简短。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */
 #navigation {}
 .atr {}

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good */
 #nav {}
 .author {}

  • ID 和 Class 命名中单词应该全部小写，单词之间使用 - 作为分隔符。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */
 #videoId {}
 .demoimage {}
 .error_status {}

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good */
 #video-id {}
 .ads-sample {}

  • 不能「MUST NOT」把 ID 和 Class 选择符作为类型选择符的限定符，这样做没必要，反而还影响性能。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */

 div.error {}

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good */

  • CSS 属性中的 0 值不应该带单位。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */

组织格式

  • 必须采用 4 个空格为一次缩进。

  • CSS 属性声明应该按字母升序排列。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good */
 background: fuchsia;
 border: 1px solid;
 -moz-border-radius: 4px;
 -webkit-border-radius: 4px;
 border-radius: 4px;
 color: black;
 text-align: center;
 text-indent: 2em;

  • CSS 每个代码块相对于父代码库必须有缩进。

     /*good*/
 @media screen, projection {

 html {
 background: #fff;
 color: #444;
 }

 }

  • CSS 属性声明必须以分号结尾。

  • CSS 属性名冒号后必须有一个空格。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */
 color:#eebbcc;

 /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 good */
 color: #ebc;

  • CSS 中的属性名建议按照字母顺序排列，可以使用 Sublime Text 的 F5 命令来自动格式化。

  • 最后的选择符与 { 之间必须有一个空格。

     /Applications /Library /Network /System /Users /Volumes /bin /cores /dev /etc /home /installer.failurerequests /net /private /sbin /tmp /usr /var /用户信息 bad */
